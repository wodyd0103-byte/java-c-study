## 🔴 1. Employee.java — 생성자 `id` 버그 + 음수 검증 누락

### 문제
```java
// 수정 전
Employee(String name, int age, String department) {
    this.id = id;   // ← BUG: id 파라미터가 없어서 자기 자신(0)을 대입
    this.name = name;
    this.age = age;  // ← BUG: 음수 검증 없음 (-5 입력 시 그대로 저장)
    this.department = department;
    totalCount++;
}
```

### 원인
- `id` 매개변수가 생성자에 없는데 `this.id = id`를 호출 → 모든 직원의 id가 **0**
- `age`에 음수 검증이 없어 `-5` 같은 값이 그대로 저장됨 → `ExceptionTest.testNegativeAge` 실패
- 생성자가 package-private이라 다른 패키지에서 상속 시 `super()` 호출 불가

### 수정 내용
```java
// 수정 후
protected Employee(String name, int age, String department) {
    this.id = totalCount + 1;              // auto-increment 적용
    this.name = name;
    this.age = (age > 0) ? age : 0;        // 음수 검증: 0으로 보정
    this.department = department;
    totalCount++;
}
```

### 변경 사항
| 항목 | 수정 전 | 수정 후 |
|------|---------|---------|
| `this.id` | `id` (항상 0) | `totalCount + 1` (auto-increment) |
| `this.age` | `age` (음수 허용) | `(age > 0) ? age : 0` (음수 보정) |
| 접근 제어자 | package-private | `protected` (다른 패키지 상속 허용) |

## 🔴 2. EmployeeDaoImpl.java — 추상 클래스 인스턴스화 에러

### 문제
```java
// 수정 전
employees.add(new Employee(1, "김자바", 30, "개발팀", 417));   // ← 컴파일 에러!
employees.add(new Employee(2, "이파이썬", 28, "데이터팀", 400)); // ← 컴파일 에러!
employees.add(new Employee(3, "박스프링", 35, "백엔드팀", 400)); // ← 컴파일 에러!
```

### 원인
- `Employee`는 **abstract 클래스** → `new Employee(...)` 불가능
- `Employee` 생성자는 **3개 파라미터**(name, age, dept)인데 **5개**를 전달

### 수정 내용
```java
// 수정 후 — FullTime 구체 클래스 사용 (이름, 나이, 부서, 연봉)
employees.add(new FullTime("김자바", 30, "개발팀", 5004));    // 월급 417만 × 12
employees.add(new FullTime("이파이썬", 28, "데이터팀", 4800));  // 월급 400만 × 12
employees.add(new FullTime("박스프링", 35, "백엔드팀", 4800));  // 월급 400만 × 12
```

## 🔴 3. EmployeeDao.java — 인터페이스 외부 접근 불가

### 문제
```java
// 수정 전 (EmployeeDaoImpl.java 내부에 정의됨)
interface EmployeeDao {  // ← package-private: 외부 패키지에서 접근 불가
    ...
}
```

### 원인
- `EmployeeDao`가 `EmployeeDaoImpl.java` 파일 내부에 package-private으로 선언됨
- `EmployClient.java`(다른 패키지)에서 `EmployeeDao`를 import할 수 없었음
- Java는 하나의 파일에 두 개의 `public` 클래스를 가질 수 없음

### 수정 내용
- `EmployeeDao.java`를 **별도 파일**로 분리
- `public interface EmployeeDao`로 접근성 향상

```java
// EmployeeDao.java (신규 파일)
package kr.ac.kopo.ctc.aisw.kopo14.model;

import java.util.List;

public interface EmployeeDao {
    Employee create(Employee employee);
    List<Employee> readOne(int id);
    List<Employee> readAll();
    Employee update(int id, Employee employee);
    Employee delete(int id);
    List<Employee> readList(int offset, int limit, String sortBy, String order);
}
```

## 🟡 4. EmployeeDaoImpl.readList — 정렬 파라미터 미사용

### 문제

```java
// 수정 전
public List<Employee> readList(int offset, int limit, String sortBy, String order) {
    return employees.stream().skip(offset).limit(limit).toList();
    // ← sortBy, order 파라미터가 완전히 무시됨!
}
```

### 수정 내용
```java
// 수정 후 — Comparator를 활용한 정렬 구현
public List<Employee> readList(int offset, int limit, String sortBy, String order) {
    Comparator<Employee> comparator = null;
    if ("name".equalsIgnoreCase(sortBy)) {
        comparator = Comparator.comparing(Employee::getName);
    } else if ("age".equalsIgnoreCase(sortBy)) {
        comparator = Comparator.comparingInt(Employee::getAge);
    } else if ("salary".equalsIgnoreCase(sortBy)) {
        comparator = Comparator.comparingDouble(Employee::calcSalary);
    } else {
        comparator = Comparator.comparingInt(Employee::getId);  // 기본: id순
    }

    if ("desc".equalsIgnoreCase(order)) {
        comparator = comparator.reversed();
    }

    return employees.stream()
            .sorted(comparator)
            .skip(offset)
            .limit(limit)
            .toList();
}
```

### 지원 정렬 기준

| `sortBy` 값 | 정렬 기준 | `order` 값 | 정렬 방향 |
|-------------|----------|-----------|----------|
| `"name"` | 이름순 | `"asc"` (기본) | 오름차순 |
| `"age"` | 나이순 | `"desc"` | 내림차순 |
| `"salary"` | 급여순 | | |
| 그 외 | id순 (기본) | | |

```java
// EmployeeManagerTest.java 수정 전
@BeforeEach

```

### 수정 내용

```java
// 추가된 메서드
public void clear() { employees.clear(); }
```

```java

    manager = EmployeeManager.getInstance();
    manager.clear();  // 테스트 간 격리: 직원 목록 초기화
}
```
